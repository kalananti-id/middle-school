APP UTAMA • DESIGNER

Pembanding Harga per Satuan

Tujuan: hitung Rp per ml/g untuk A dan B, lalu tampilkan keputusan.

Komponen wajib
  • • TextBox NumbersOnly: tbPriceA, tbQtyA
  • • TextBox NumbersOnly: tbPriceB, tbQtyB
  • • Button: btnCompare
  • • Label: lblUnitA, lblUnitB, lblWinner
  • • Notifier: Notifier1
Wajib
Qty/ukuran tidak boleh 0 (kalau 0 → Notifier).

APP UTAMA • BLOCKS

Logika Pembanding Harga

Langkah: validasi → hitung unit → bandingkan → tampilkan keputusan.

1) Validasi + hitung unit
Blocks (gambar)
Menghitung harga per gram untuk barang 1 dan 2, lalu menampilkan ke label.
2) Bandingkan (if/else)
Blocks (gambar)
Membandingkan hasil harga per gram untuk menentukan barang yang lebih murah.
Tes wajib
A: 5000/1000=5, B: 3000/500=6 → A lebih hemat

Checklist akhir (Pembanding Harga)
  • • Ada validasi input kosong.
  • • Ada validasi qty tidak boleh 0.
  • • Unit A & Unit B tampil.
  • • Keputusan tampil (A/B hemat/sama).
  • • Sudah tes dengan 2 contoh berbeda.

RINGKASAN

  • • Logika finansial itu urutan langkah: identifikasi → hitung → bandingkan → putuskan.
  • • Simulasi tabungan: total = daily × days, lalu cek total ≥ target.
  • • App Inventor memudahkan kita membuat kalkulator sederhana untuk membantu keputusan.
  • • Proyek utama: Pembanding Harga menghitung harga per satuan agar pilih yang paling hemat.
  • • Checklist aman: input tidak kosong, qty tidak 0, hasil jelas (angka + kesimpulan).
