4 Pilar Validasi Input

01. Presence Check

Apakah ada isinya?

Mencegah data kosong (null/empty). Penting untuk field wajib seperti Nama atau Email.

If Textbox1.Text is Empty Then ...
02. Type Check

Apakah tipenya benar?

Memastikan input adalah Angka (Whole Number) untuk field umur, harga, atau jumlah.

If isNumber(Textbox1.Text) Then ...
03. Length Check

Apakah panjangnya pas?

Password minimal 8 karakter. Kode Pos harus 5 digit. Tweet max 280 char.

If Length(Textbox1.Text) < 8 Then ...
04. Range Check

Apakah masuk akal?

Nilai harus dalam batas wajar. Umur manusia 0-120. Nilai ujian 0-100.

If Score < 0 OR Score> 100 Then ...

1. The "Guard" Block

Kita menggunakan blok kontrol IF-THEN-ELSE. Jika syarat terpenuhi (True), lakukan A. Jika tidak (Else), lakukan B (Error).

  // Data Aman, Lanjut
  OpenScreen("Home")
ELSE
  // Data Salah, Stop

Kenapa pakai OR?

Kita ingin memblokir jika SALAH SATU saja kosong.
Username kosong? Block.
PIN kosong? Block.
Jadi pakai logika OR.

Tantangan!

Coba cari blok "OR" di menu Logic (warna hijau muda) dan blok "is empty" di menu Text (warna pink).
