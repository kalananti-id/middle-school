Pemanasan: Mengingat Kembali Kondisi Dasar

Sebelum belajar kondisi kompleks, yuk kita ingat dulu apa itu if-then-else.

Contoh Sederhana:

Tapi... bagaimana kalau ada lebih dari satu kondisi yang perlu dicek?

Misalnya, kita ingin tahu apakah suhu udara panas, sedang, atau dingin. Nah, di sinilah kita butuh kondisi kompleks!

Apa Itu Kondisi Kompleks?

Kondisi kompleks adalah saat program harus memeriksa beberapa kondisi secara berurutan.

Bayangkan kamu punya aturan seperti ini:

  • üåü Jika nilai di atas 90 ‚Üí "Sangat Baik"
  • üòä Jika nilai di atas 80 ‚Üí "Baik"
  • üëç Jika nilai di atas 70 ‚Üí "Cukup"
  • üìö Selain itu ‚Üí "Perlu Belajar Lagi"

Kita tidak bisa pakai satu blok if saja, karena keputusan tergantung pada banyak hal.

Jadi, kita akan menggunakan if di dalam if ‚Äî itulah yang disebut nested condition (kondisi bersarang).

Struktur Kondisi Bersarang

Secara umum, bentuk logika kondisi kompleks bisa digambarkan seperti ini:

if (kondisi 1 benar) lakukan A
else if (kondisi 2 benar) lakukan B
else if (kondisi 3 benar) lakukan C
else lakukan D

Menggunakan Kondisi di AppInventor

Di AppInventor, blok yang digunakan untuk membuat kondisi kompleks adalah:

  • if then ‚Üí memeriksa satu kondisi
  • if then else ‚Üí menambahkan pilihan lain jika kondisi tidak benar
  • else if ‚Üí menambah kondisi tambahan di antara keduanya

Langkah Umum:

  1. Tambahkan blok kontrol if-then-else
  2. Tambahkan kondisi menggunakan blok perbandingan (=, >, <, ‚â•, ‚â§)
  3. Gunakan blok else if jika ada lebih dari dua kemungkinan
  4. Tempatkan aksi atau hasil di dalam bagian then atau else
