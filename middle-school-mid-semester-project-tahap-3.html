Tahap 3: Penyusunan Blok Kode & Logika Program

Pada Tahap 2, kamu telah menyelesaikan desain tampilan aplikasimu: membuat screen, menambahkan komponen UI, serta menata layout agar rapi dan mudah digunakan.

Sekarang di Tahap 3, kamu akan mulai membuat blok kode di MIT AppInventor untuk menghidupkan aplikasi. Pada tahap ini, kamu akan:

  • Menghubungkan komponen UI dengan event yang sesuai (misalnya tombol untuk menjalankan perintah).
  • Menggunakan variabel untuk menyimpan dan mengolah data keuangan.
  • Menyusun operasi matematika sesuai kebutuhan aplikasi.
  • Menerapkan percabangan if–then–else untuk membuat keputusan dalam program.
  • Menguji aplikasi dan memperbaiki error sederhana (debugging).

Hasil Tahap 3 ini akan menjadi dasar untuk Tahap 4, yaitu proses finalisasi dan presentasi proyek.

Tujuan Pembelajaran Tahap 3

Setelah menyelesaikan Tahap 3, kamu diharapkan mampu:

  • Menghubungkan komponen UI dengan blok event yang sesuai (misalnya tombol dengan perintah tertentu).
  • Menggunakan variabel untuk menyimpan dan mengolah data (saldo, pemasukan, pengeluaran).
  • Mengimplementasikan logika aplikasi sesuai flowchart yang kamu buat di Tahap 1.
  • Menyusun blok operasi matematika dan percabangan if–then–else.
  • Melakukan uji coba (testing) dan memperbaiki error sederhana (debugging).

Instruksi Assignment Tahap 3

Sebelum mulai menyusun blok kode, pastikan kamu sudah membuka assignment resmi di LMS.

Worksheet Tahap 3 berisi bagian-bagian yang perlu kamu lengkapi: daftar blok kode penting, penjelasan alur logika, screenshot blok, dan laporan debugging.

Variabel & Event di AppInventor

Variabel digunakan untuk menyimpan nilai yang dapat berubah-ubah, misalnya:

  • global Saldo – menyimpan total uang yang dimiliki.
  • global TotalPemasukan – menyimpan total semua pemasukan.

Event adalah “pemicu” kapan kode dijalankan, misalnya:

  • when Button_Tambah.Click – kode dijalankan saat tombol tambah ditekan.
  • when Screen.Initialize – kode dijalankan saat screen pertama kali dibuka.

Di Tahap 3, kamu akan menghubungkan variabel dan event ini dengan logika perhitungan dalam aplikasi keuanganmu.

Contoh: global Saldo – menyimpan jumlah uang yang dimiliki; when Button_Spend.Click – memproses pengeluaran dan mengurangi saldo.

Menjelaskan Alur Logika Program

Selain daftar blok, kamu juga perlu menuliskan bagaimana alur logika aplikasimu bekerja menggunakan kalimat sederhana.

Gunakan panduan berikut di worksheet:

Template · Penjelasan Alur Logika

Ketika tombol __________________ ditekan, aplikasi akan __________________________.

Jika pengguna memasukkan nilai __________________, maka ________________________.

Aplikasi akan menampilkan _________________________________ pada screen __________.

Tulislah sesuai dengan proyekmu. Pastikan penjelasanmu sesuai dengan blok kode yang kamu buat.

Debugging: Mencatat dan Memperbaiki Error

Error atau bug adalah hal yang wajar saat membuat program. Di Tahap 3, kamu diminta menuliskan minimal satu masalah yang kamu temui dan bagaimana kamu memperbaikinya.

Template · Debugging Report
Masalah / Error Penyebab Cara Saya Memperbaikinya
__________________ __________________ __________________
__________________ __________________ __________________

Contoh: “Saldo tidak bertambah” – penyebab: lupa mengubah TextBox.Text menjadi angka – solusi: menambahkan blok convert to number.

Screenshot Blok Kode & Testing

Sebagai bagian dari dokumentasi, kamu diminta mengambil beberapa screenshot blok kode, misalnya:

  • Blok variabel global (misalnya global Saldo).
  • Blok event tombol (misalnya when Button_Add.Click).
  • Blok if–then–else untuk pengecekan saldo cukup atau tidak.

Saat melakukan screenshot:

  • Pastikan blok terlihat jelas dan tidak terpotong.
  • Berikan nama file yang mudah dikenali.
  • Tempel screenshot pada bagian yang disediakan di worksheet Tahap 3.

Jangan lupa melakukan testing setiap kali kamu menambah atau mengubah blok kode, supaya lebih mudah menemukan error sejak awal.
